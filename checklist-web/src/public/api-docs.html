ChecklistHub API Documentation

REST API reference for the Expo mobile client and other JSON clients. All examples assume the API is served from the same host as the Next.js app.

Authenticated endpoints accept Authorization: Bearer <jwt>. JSON list endpoints return { "data": [], "pagination": {} }.

Conventions

Content Type

Send request bodies as application/json. Responses are JSON except this docs page.

Authentication

Call POST /api/auth/login and store the returned JWT token in the mobile app.

Pagination

Use page and pageSize. Invalid values fall back to endpoint defaults.

Errors

Errors use { "error": "Message" } with an appropriate HTTP status code.

Pagination DTO

{
  "data": [],
  "pagination": {
    "page": 1,
    "pageSize": 10,
    "total": 42,
    "totalPages": 5
  }
}

Authentication

POST /api/auth/register

Creates a user account and returns a JWT token.

Request

{
  "name": "Alex Diver",
  "email": "alex@example.com",
  "password": "pass123"
}

Response 201

{
  "token": "jwt-token",
  "user": {
    "id": 1,
    "email": "alex@example.com",
    "name": "Alex Diver",
    "role": "user"
  }
}
POST /api/auth/login

Logs in by email and password. Store the returned token in secure mobile storage.

{
  "email": "alex@example.com",
  "password": "pass123"
}
GET /api/auth/me

Returns the current authenticated user.

Authorization: Bearer jwt-token

Templates

GET /api/templates

Lists published templates. Supports pagination and optional filters.

page: default 1 pageSize: default 10, max 50 q: search title/description category: category slug 
GET /api/templates?page=1&pageSize=10&q=scuba&category=diving
GET /api/templates/[id]

Returns template metadata, category, activity, sections, and official items.

Checklists

Checklists are private. A user can only list, read, or modify their own checklists.
GET /api/checklists

Lists the authenticated user's checklists with progress information.

GET /api/checklists?page=1&pageSize=10
Authorization: Bearer jwt-token
POST /api/checklists

Creates a personal checklist copy from a published template.

{
  "templateId": 12
}
GET /api/checklists/[id]

Returns checklist details, sections, user-editable items, and progress.

POST /api/checklists/[id]/items

Adds a custom item to one section of an owned checklist.

{
  "sectionId": 34,
  "text": "Pack spare battery"
}
POST /api/checklists/[id]/items/[itemId]/toggle

Toggles an item between complete and incomplete.

Suggestions

GET /api/suggestions

Lists suggestions submitted by the authenticated user, including admin notes and comments.

POST /api/suggestions

Creates a proposal for new activity/template work or improvements to an existing template.

{
  "type": "template_edit",
  "title": "Add emergency oxygen check",
  "description": "The scuba pre-dive list should include oxygen availability.",
  "targetTemplateId": 12
}

Allowed type values: new_activity, new_template, template_edit, template_variant. Template edit and variant suggestions require targetTemplateId.
